# GROUND TRUTH — rung 3 (MULTI-finding; NEVER fed to the finder; scorer-only)

Three planted findings of different classes. Precision is honest here: a lead is "noise" only if it
matches NONE of these three. G4 (round-trip) is a clean control — there is no bug there.

## Finding 1 — access control: `setRewardRate` is not admin-gated  (HIGH)
**Location:** `dreStaking::setRewardRate`. **Promise broken:** G2 (admin-only economics).
**Mechanism:** every other economics function (`fundRewards`) has `onlyAdmin`, but `setRewardRate` has no
modifier. Any address can set `rewardRate` to an arbitrary value. **Exploit:** attacker stakes 1 wei, calls
`setRewardRate(huge)`, waits, then `claim()` — `earned()` pays out against the inflated rate and drains
`reserve` (and other stakers' funds). **Fix:** add `onlyAdmin` to `setRewardRate`.

## Finding 2 — reward accounting: `fundRewards` drops the un-streamed leftover  (HIGH)
**Location:** `dreStaking::fundRewards`. **Promise broken:** G1 (conservation of rewards).
**Mechanism:** Synthetix-style funding must roll the current period's undistributed rewards into the new
rate: `leftover = (periodFinish - block.timestamp) * rewardRate; rewardRate = (amount + leftover)/DURATION`.
The code uses `rewardRate = amount / DURATION`, discarding `leftover`. Any rewards from an unfinished period
are stranded in `reserve` and never stream to stakers — funded reward silently dropped. **Fix:** add the
leftover term when recomputing `rewardRate`.

## Finding 3 — rounding direction: `earned()` rounds UP, favoring the user  (MEDIUM)
**Location:** `dreStaking::earned`. **Promise broken:** G3 (rounding favors the protocol).
**Mechanism:** `(staked[who] * delta + (1e18 - 1)) / 1e18` is a ceiling division — it rounds the reward UP,
crediting the staker up to ~1 wei more than owed on every accrual. Per G3 it must round DOWN (`/ 1e18` with
no `+1e18-1`). Aggregated across many stakers/accruals the reserve is over-drawn; combined with `claim`'s
`reserve -= r`, the last claimants can be blocked (underflow revert). **Fix:** drop the `+ (1e18 - 1)`.

**Difficulty note:** harder than rung 2 — three bugs of distinct classes (authorization, streamed-accounting
carry-over, rounding direction), each requiring a different reasoning mode, in a realistic Synthetix-pattern
contract where the buggy lines look idiomatic. Tests breadth (does the finder cover all three classes?) and
honest precision (noise = leads matching none of the three).
